# Competitive Landscape: AI Coding Tool Companions

## Summary

No major AI coding tool has a built-in, agent-aware companion comparable to Claude Code's `/buddy`. Decorative pet extensions exist (VS Code Pets, Codachi) but are disconnected from the coding agent's internal state.

## Positioning

Claude Code's buddy occupies the **missing middle**:
- **Embodied like a pet** (visual presence, personality, name)
- **Functional like an agent** (sees tool output, reacts to code changes)
- **Customizable like a plugin platform** (proposed)

### Defensible claim

> "Competitors let users customize how coding agents think and act through prompts, rules, tools, and MCP integrations, but no major AI coding tool exposes a built-in companion as a customizable, agent-aware plugin surface. Decorative pet extensions exist but are disconnected from the assistant's actual tool loop and coding context."

### What competitors DO offer (and we're not competing with)

All major tools offer behavioral customization:
- Custom instructions/prompts
- Rules files
- MCP/tool extensibility
- Agent configuration

What they DON'T offer: a **visual, ambient, personality-driven companion** that's connected to the agent's observation stream. That's the gap.

## What makes this different from agent plugins

Companion plugins don't customize how the agent thinks — they customize the ambient presence that watches and reacts. Different art, different voice, different triggers, different feedback style. It's a new layer on top of the existing plugin system, not a replacement for any of it.
